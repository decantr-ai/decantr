# Blueprint Showcase System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a live demo showcase system where each blueprint in the Decantr registry has a navigable, interactive demo — scaffolded via `decantr init`, built with Vite + React + `@decantr/css`, and served as static files from the existing Vercel deployment.

**Architecture:** Standalone Vite + React projects under `apps/showcase/{blueprint}/`, each scaffolded with `decantr init` from a blueprint. A build step compiles each to static HTML/CSS/JS, copies output to `apps/web/public/showcase/{name}/`, and serves them from the same Vercel deployment. The web app's registry detail page links to demos.

**Tech Stack:** Vite, React 19, React Router, `@decantr/css`, `@decantr/vite-plugin`, pnpm workspaces

**Spec:** `docs/specs/2026-03-31-blueprint-showcase-design.md`

```tsx
import { useState } from 'react';
import { showcases, currentShowcase } from './showcase-registry';

export function ShowcaseChrome({ children }: { children: React.ReactNode }) {
  const [dropdownOpen, setDropdownOpen] = useState(false);

  return (
    <div style={{ display: 'flex', flexDirection: 'column', minHeight: '100vh' }}>
      <header
        style={{
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'space-between',

            textDecoration: 'none',
            fontWeight: 500,
          }}
        >
          ← decantr.ai
        </a>

        <div style={{ position: 'relative' }}>
          <button
            onClick={() => setDropdownOpen(!dropdownOpen)}
            style={{
              background: 'var(--d-surface, #1a1a1a)',
              border: '1px solid var(--d-border, #333)',
              borderRadius: 'var(--d-radius, 9999px)',
              color: 'var(--d-fg, #fafafa)',
              padding: '4px 12px',
              cursor: 'pointer',
              fontSize: '13px',
            }}
          >
            {currentShowcase.name} ▾
          </button>

          {dropdownOpen && (
            <div
              style={{
                position: 'absolute',
                right: 0,
                top: '100%',
                marginTop: '4px',
                background: 'var(--d-surface, #1a1a1a)',
                border: '1px solid var(--d-border, #333)',
                borderRadius: '8px',
                minWidth: '220px',
                zIndex: 100,
                overflow: 'hidden',
              }}
            >
              {showcases.map((s) => (
                <a
                  key={s.slug}
                  href={`/showcase/${s.slug}/`}
                  style={{
                    display: 'block',
                    padding: '8px 12px',
                    color:
                      s.slug === currentShowcase.slug
                        ? 'var(--d-primary, #3b82f6)'
                        : 'var(--d-fg, #fafafa)',
                    textDecoration: 'none',
                    fontSize: '13px',
                  }}
                  onClick={() => setDropdownOpen(false)}
                >
                  <div style={{ fontWeight: 500 }}>{s.name}</div>
                  <div
                    style={{
                      fontSize: '11px',
                      color: 'var(--d-fg-muted, #999)',
                    }}
                  >
                    {s.theme}
                  </div>
                </a>
              ))}
            </div>
          )}
        </div>
      </header>

      <div style={{ flex: 1, display: 'flex', flexDirection: 'column' }}>
        {children}
      </div>
    </div>
  );
}
```

Note: Inline styles are used intentionally here instead of `@decantr/css` atoms. The showcase chrome is a meta-layer OUTSIDE the demo's theme — it should not be affected by the blueprint's theme tokens. The CSS custom property fallbacks (`var(--d-bg, #0a0a0a)`) provide sensible defaults.

The specific code for each shell depends on the DECANTR.md output and the shell specs retrieved in Step 2. Follow those exactly rather than inventing layout structures.

## Task 9: Copy Script

Build the script that copies showcase build output to the web app's public directory.

**Files:**
- Create: `scripts/copy-showcase-dist.js`

- [ ] **Step 1: Create the copy script**

```js
// scripts/copy-showcase-dist.js

import { readdirSync, existsSync, cpSync, mkdirSync, rmSync } from 'node:fs';
import { join, basename } from 'node:path';

const showcaseDir = join(import.meta.dirname, '..', 'apps', 'showcase');
const targetDir = join(import.meta.dirname, '..', 'apps', 'web', 'public', 'showcase');

// Clean target directory
if (existsSync(targetDir)) {
  rmSync(targetDir, { recursive: true });
}
mkdirSync(targetDir, { recursive: true });

// Find all showcase projects with a dist/ directory
const projects = readdirSync(showcaseDir, { withFileTypes: true })
  .filter((d) => d.isDirectory())
  .filter((d) => existsSync(join(showcaseDir, d.name, 'dist')));

if (projects.length === 0) {
  console.log('No showcase builds found. Run "pnpm showcase:build" first.');
  process.exit(0);
}

for (const project of projects) {
  const src = join(showcaseDir, project.name, 'dist');
  const dest = join(targetDir, project.name);
  cpSync(src, dest, { recursive: true });
  console.log(`  ✓ ${project.name} → public/showcase/${project.name}/`);
}

console.log(`\nCopied ${projects.length} showcase(s) to apps/web/public/showcase/`);
```
