Design Section 3: apps/workbench

Local dev tool for building, testing, and exploring @decantr/ui components

Three Views (phased delivery)

Phase 1 — Priority

Component Isolation

Select a component from the sidebar. See all its variants rendered in a grid. Switch themes, density, and mode. Components live-reload as you edit source files.

  • Variant grid from story definitions
  • Theme/density/mode toolbar
  • Hot-reload via the Decantr compiler dev mode
  • CSS inspector panel
Phase 2

Playground

Interactive prop editor driven by the story's playground controls. Tweak every prop, see the component update live. Copy the resulting code snippet.

  • Auto-generated controls from story schema
  • Live component preview
  • Code output panel (copy-paste ready)
  • Signal state inspector
Phase 3

Design System Explorer

Bird's-eye view of the full system. Theme gallery, density comparison, accessibility audit, color palette visualization, typography scale, spacing scale.

  • All themes side-by-side
  • Density level comparison (same component, 3 densities)
  • Colorblind simulation modes
  • WCAG contrast checker per theme
  • Full icon gallery with search

Technical Details

  • Built with @decantr/ui — the workbench itself is a Decantr-native app (dog-fooding the framework's router, SSR, state management)
  • Dev server — uses the Decantr compiler's dev mode (compiler/dev.js) for file watching + hot-reload
  • Local only — runs on localhost, no deployment needed. pnpm --filter workbench dev
  • Reads catalog — imports @decantr/ui-catalog to get story definitions, renders them with its own chrome
  • No persistence needed — the workbench is stateless. Components render from source, stories define what to show
